# 第 1 章：认识 XIRASM

## 同一份源文件，两种写法

XIRASM 源文件可以包含两种形式的代码：

- **ISA 文本**：描述所选处理器的指令，保持常规汇编写法。
- **编译期代码**：在汇编过程中计算值、生成输出，并控制源码如何被组装。

它们属于同一种源语言。

编译期值可以直接出现在指令中，标号可以交给布局 API 使用，函数也可以同时生成数据和 ISA 文本。

例子：

```asm
// 开启 64 位 x86 模式。
x86.use64();

// 编译期计算常量，指令里直接用。
const answer: u32 = 40 + 2

entry:
    // 把计算好的数值放进 eax，然后返回。
    mov eax, answer
    ret
```

简单说明：

- `x86.use64();` 选择 64 位 x86 指令编码模式。
- `const answer: u32 = 40 + 2` 声明编译期常量。
- `entry:` 在当前逻辑地址定义一个标号。
- `mov eax, answer` 是普通 x86 指令，把 `answer` 移入 eax，编译期自动代入常量值。
- `ret` 是普通 x86 指令。

`x86.use64();` 这类调用使用结构化的编译期语法。`mov eax, answer` 仍然是自然 ISA 文本，不需要写成函数调用。

## 运行一个例子

把源码保存为 `hello.xir`，然后执行：

```text
xirasm hello.xir -o hello.bin --target x86-64
```

输出字节：

```text
b8 2a 00 00 00 c3
```

前五个字节是 `mov eax, 42` 的编码，最后一个字节是 `ret`。标号不产生字节，只记录一个地址，供其他指令和编译期 API 使用。

直接汇编时，XIRASM 默认写出 flat binary，也就是源码实际生成的字节序列；它不带操作系统文件头。后面的章节会介绍格式接口，完整的 PE、COFF 和 ELF 程序见格式教程。

## 编译期代码生成输出

编译期代码在汇编过程中执行，不是在最终程序里运行。

比如循环写四个字节：

```asm
// 编译期循环四次，每次写一个字节。
const count: u8 = 4

for value in range(0, count) {
    db(value);
}
```

生成的字节：

```text
00 01 02 03
```

`range(0, count)` 生成从 0 到 count-1 的序列。`for` 在编译期执行，每次调 `db(value)` 写一个字节。

这个区别很关键：

| 做什么           | 怎么写           |
| ---------------- | ---------------- |
| 手写 ISA 指令    | ISA 文本         |
| 计算和判断       | 编译期表达式     |
| 重复或条件生成   | 编译期控制流     |
| 输出字节         | `db`、`dw`、`dd`、`dq` 等数据调用 |
| 封装可复用逻辑   | 函数             |
| 生成 PE/COFF/ELF | 对应的格式接口   |

## 源文件语法注意点

几条简单规则：

- `//` 到行尾是单行注释，也可以写在 ISA 指令后；ISA 字符串操作数中的 `//` 仍是普通文本。
- ISA 指令行不以分号结尾。
- 标号以 `:` 结尾。
- 函数和接口调用以 `;` 结尾。
- `return` 语句用 `;` 结尾。
- `const` 和 `let` 声明末尾**不加**分号。
- 代码块用 `{` `}`。
- 调用和返回值可以跨行，左括号会一直等到对应的右括号。

完整例子：

```asm
// 定义编译期可调用的辅助函数。
fn emit_marker(value: u8) {
    // 写固定标记，后面跟传入的值。
    db(0x7f, value);
}

const marker: u8 = 3
emit_marker(marker);

start:
    // ISA 指令里可以直接使用编译期常量；指令行末尾不写分号。
    mov eax, marker
    ret
```

函数定义和函数调用属于编译期代码；标号和下面两行缩进的指令属于 ISA 文本。两种写法共同生成同一个最终输出。

## 怎么读 XIRASM 源码

读源码时，按这几步判断每行：

1. 如果是 ISA 指令，就写自然 ISA 文本。
2. 如果要在汇编期间计算、选择或重复生成内容，就使用编译期语言。
3. 如果要写出字节或预留空间，就使用数据和输出 API。
4. 如果要描述完整文件格式，就使用对应的格式接口，不要手算文件头表项。
